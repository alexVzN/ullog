# Security Policy

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security-sensitive reports.

Use GitHub's [Private Vulnerability Reporting](https://github.com/alexVzN/ullog/security/advisories/new)
to file a private security advisory. The maintainer will receive your report
privately and can coordinate a fix with you before any public disclosure.

You can expect an initial response within a few business days.

## Supported Versions

Only the latest released version on `main` receives security fixes.
